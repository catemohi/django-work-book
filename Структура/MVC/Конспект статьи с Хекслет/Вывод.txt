По истогу конспекта:
MVC - это архитектурный паттерн, способ организации кода, при котором выделяются 3 компонента кода, каждый со своей задачью.

Компоненты и их задачи:
Модель - Компонент отвечающий за данные и их сохранение, получение.
Представление - Компонент отвичающий за взаимодействие с пользователем. Готовый ответ, который будет получен на запрос.
Контроллер - Компонент отвечающий за логику и связь между моделью и представлением. Получает данные из модели и формирует их для представления.

Можно сказать, что:
Контроллер обрабатывает входящие запросы. Модель достаёт из базы данных информацию, нужную для выполнения конкретных запросов. Представление определяет результат запроса, который получает пользователь.
